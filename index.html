Features

High level Vespa features.

Large-scale applications, 100k queries per second, tens of billions of documents.

Your large-scale applications require scalability and efficiency. Vespa scales linearly with the number of machines in terms of data storage, query traffic, and feeding capability. 100k queries per second and tens of billions of records are within normal use cases. Efficiency minimizes the costs associated with resources in a large-scale system (machine cost, cooling, power, maintenance, security, and operation). Vespaâ€™s search core is written in well-profiled C++ code to be extremely efficient. Algorithms are carefully chosen and developed, while performance and efficiency are monitored.

Real-time. UI friendly response times.

Use Vespa directly within your UI, with latencies of only tens of milliseconds. Data discovery occurs in real time, immediately after feeding.

Live upgrades and reconfiguration. No need for down time.

You can change the configuration of your system without taking it down or duplicating it. Vespa is designed and built for 24/7 availability, including during upgrades. You can even make changes to your document structure while serving.

Configurable redundancy. The redundancy you want.

By tuning redundancy, you can set up the solution that works for you in terms of hardware cost, storage capacity, and more.

Native machine learning, deep learning, and advanced retrieval support.

Supports machine learning models for serving advanced retrieval logic. Vespa has powerful search features, including grouping and phased ranking. The solution supports various types of search strategies, including index-based and personal searches (streaming data search).

Fault-tolerant. Data is automatically redistributed to new machines.

When redundancy is higher than 1, Vespa will redistribute the data from a broken machine among all nodes using optimal algorithms, with minimal changes to data distribution while the cluster is serving traffic. When the node is replaced, Vespa will redistribute the data again. This fully-automatic process preserves all of your data.

Battled-proofed. Vespa is a proven solution currently used throughout Yahoo for mail search, ad serving, and rendering personalized Yahoo front pages.

Vespa is a proven solution currently used throughout Yahoo for mail search, ad serving, and rendering personalized Yahoo front pages. It is a result of tens of years of continuous development and operation. We have sophisticated internal test and build pipelines, as well as canary applications.

Supports modern infrastructure. Runs on Docker, CentOs and more.

Runs in data centers (Cent OS) and Linux platforms directly, and also supports Docker. Vespa works with Ip6/Ip4, and requires only a single machine to run.

Simple APIs. JSON REST APIs.

Vespa APIs are based on REST/JSON. We offer a Java client library to aid in auto-throttling and high-performance feeding. The library makes it easy to feed from hundreds of grid machines to a small cluster that might be serving live UI traffic.

Flexible. Many power user features supporting lots of use cases.

Accepts Java code and OSGI when you want full control. Write your own custom data processing logic in Java that rewrites queries, and pre-process documents. The code can be reconfigured while the system is fully serving traffic. Expert users can also take advantage of very advanced messaging system.
